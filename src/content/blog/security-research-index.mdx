This is a deliberately opinionated index for Windows security research and
exploit development. It prioritizes original research, reproducible technical
work, source code, debugger-level explanations, and material that remains useful
after a specific campaign or Windows build ages out. The focus spans BYOVD,
kernel exploitation, COM/RPC/ALPC, APC, loaders, injection, malware internals,
mitigations, EDR, and low-level references.

The researcher list is not a popularity ranking. It favors people and teams with
a sustained body of original Windows work; the profile link points to X or
GitHub where useful, while the blog link points to the material worth reading.
Search filters every table at once, including hidden topic keywords.

The **Year** column records the publication or initial project year where it can
be established from the original source. **Living** means the page or repository
is an actively maintained index rather than a single dated article.

The stars are an editorial **reading priority**, not GitHub popularity. Five
stars marks foundational or unusually complete material; four marks strong
practical depth; three marks useful but specialized material; and two marks a
supplemental or secondary source that should be verified against primary
research. Ratings measure usefulness for this study path, not author quality.
New entries must normally earn at least four stars: primary-source research,
meaningful technical depth, and enough evidence or implementation detail to
verify the claims. Lower-rated legacy entries are retained only when they
provide context for a previously tracked lead and are not core reading.

## Scope and attribution

Descriptions are navigation notes, not substitutes for the original research.
Follow the source links for technical details and credit. Publication years and
build-specific claims should be rechecked when a project or article is updated;
Windows internals and exploitation techniques age quickly.
